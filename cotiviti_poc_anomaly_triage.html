Agentic Claims Anomaly Triage System

A three-layer pipeline that clusters providers into behavioral baselines, flags statistical outliers, and uses an agentic AI layer to explain each flag in plain language with a recommended action — combining unsupervised pattern recognition with agentic AI. An agentic AI system here refers to a large language model equipped with planning and tool-use capabilities that executes a sequence of reasoning and validation steps before generating a recommendation.

1

Clustering engine

K-Means groups 84 providers by specialty, cost, and procedure volume to establish behavioral baselines.

2

Anomaly detection

Distance-from-centroid scoring flags providers whose billing pattern deviates sharply from baseline.

3

Agentic AI layer

The agent reasons step-by-step through each flag and recommends approve, escalate, or deny.

Data: 84 synthetic provider profiles (5 specialties), with 4 deliberately planted anomalies (upcoding, phantom billing volume, extreme cost outlier, procedure count outlier) to validate the pipeline. Clustering and anomaly scoring run entirely client-side in JavaScript (K-Means + z-score distance). The agentic AI layer calls the Claude API (claude-sonnet-4-6) to generate an auditable, multi-step explanation for each flag — this is the agent described in the written report, which orchestrates verification steps before producing an approve, escalate, or deny recommendation. Human reviewer decisions feed back into the system as a retraining signal — continuously improving cluster baselines over time (feedback loop ↻ — see apply_feedback() in pipeline.py).
